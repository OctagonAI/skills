## Analyzing Question Themes

### Theme Extraction Framework

```
Step 1: List all analyst questions
Step 2: Group by topic (financial, strategic, operational, risk)
Step 3: Count frequency per topic
Step 4: Assess management response quality
Step 5: Identify unresolved concerns
```

## Question Intent Analysis

### Decoding Question Types

| Question Type | Example | Intent |
|---------------|---------|--------|
| Clarification | "Can you quantify..." | Model inputs |
| Strategic | "How do you think about..." | Direction |
| Concern | "Are you worried about..." | Risk probe |
| Comparison | "Relative to peers..." | Positioning |
| Challenge | "But you said last quarter..." | Accountability |

### Reading Between the Lines

| Question Pattern | What It Really Means |
|------------------|----------------------|
| "Help us understand..." | "We're skeptical" |
| "I just want to clarify..." | "That seems wrong" |
| "Some investors are asking..." | "I'm concerned about" |
| "Looking out longer term..." | "Short-term looks weak" |

## Response Quality Assessment

### Grading Management Responses

| Grade | Criteria | Example |
|-------|----------|---------|
| A | Direct, quantified, comprehensive | "Revenue will grow 15-18%..." |
| B | Addressed, directional | "We expect improvement..." |
| C | Partial, vague | "We're working on that..." |
| D | Deflected, avoided | "As we said earlier..." |
| F | Contradicted, hostile | Dismissive response |

### Red Flags in Responses

| Red Flag | What It Signals |
|----------|-----------------|
| "I'll let [CFO] answer" | CEO avoiding topic |
| "As we discussed" | Refusing new detail |
| "It's complex" | Obfuscating |
| Long non-answer | Hiding something |
| Attacking premise | Defensive posture |

## Concern Resolution Tracking

### Resolution Status Framework

| Status | Indicators | Action |
|--------|------------|--------|
| Resolved | Specific answer, analyst satisfied | Monitor |
| Partially Resolved | Some detail, gaps remain | Dig deeper |
| Deflected | Topic changed, general answer | Flag concern |
| Unresolved | No real answer, pushback | High priority |

### Follow-up Analysis

| Follow-up Type | Meaning |
|----------------|---------|
| Analyst pushes back | Not satisfied |
| Analyst accepts | Answer adequate |
| Different analyst same topic | Widespread concern |
| Topic dropped | Issue less important |

## Action Framework

### Post-Analysis Steps

1. **Prioritize concerns** by frequency and analyst weight
2. **Track resolution** status from management
3. **Compare to thesis** and update risk assessment
4. **Monitor** for pre-announcement or update
5. **Cross-reference** with SEC filings
6. **Anticipate** next quarter's questions
